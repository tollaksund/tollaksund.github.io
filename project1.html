Autonomous Warehouse Robot

A mobile robot designed to navigate and operate in a dynamic warehouse environment.

Overview

This robot was developed to autonomously navigate a warehouse, detect obstacles, and deliver items. It uses angle-based navigation derived from fixed landmarks (such as packing bays and shelves), paired with a **finite state machine** and real-time sensor input processed by a **model-based reflex agent**.

Design Process

The chassis was designed in **Fusion 360** and 3D-printed. The electronics were prototyped using an Arduino-based system, and the robot was tested in multiple indoor setups. The robot logic was implemented on a **Raspberry Pi** running a modular navigation system.

Key Features

  • Model-based reflex navigation system
  • Vision-based item detection and alignment
  • Obstacle avoidance using **Bug 2** and **potential fields**
  • Servo-controlled item collection and delivery
  • Modular and maintainable chassis and code architecture

System Architecture

The robot operates using a structured navigation system divided into three stages: locating the item, grabbing it, and delivering it to the packing bay. Each stage includes substages and is supported by a **finite state machine** that ensures smooth transitions and recovery if a task fails. The robot uses a model-based reflex agent to make decisions based on both current and past sensor input, improving reliability in real-time operation.

Navigation and Obstacle Avoidance

The navigation system converts sensor inputs into motor commands, enabling the robot to turn, drive straight, or correct its path. A hybrid method is used: **potential fields** for smooth long-range movement, and **Bug 2** for close-range obstacle avoidance. This allows the robot to maintain a smooth trajectory while dynamically responding to its environment.

  • Bug 2 Algorithm: Follows the edge of an obstacle until a clear path to the goal is found, ensuring reliable obstacle circumvention.
  • Potential Fields: Obstacles act as repelling forces and the goal as an attractor, resulting in fluid motion toward the target.

Operational Stages

Stage 1 – Item Location: The robot scans for the packing bay and row markers. It avoids blind zones and aligns using shelf angles to reach a stable scanning zone.

Stage 2 – Item Pickup: It raises the grabber, aligns with the detected item, and drives forward to activate the pick-up mechanism.

Stage 3 – Delivery: The robot navigates back toward the packing bay using shelf alignment and vision input. Upon arrival, it drops the item and resets the FSM.

Performance and Validation

During real-world tests, the robot successfully retrieved and delivered items with a success rate of approximately **75%**. It demonstrated robust integration between navigation, obstacle avoidance, and vision systems. Variability in lighting, battery level, and floor traction occasionally impacted reliability but did not prevent task completion.

Reflection

This project helped develop skills in **embedded programming**, **sensor integration**, and **mechanical design**. It also emphasized the importance of structured architecture, real-time decision-making, and robust testing. The model-based reflex agent combined with finite state logic proved effective in coordinating complex autonomous behavior in a constrained environment.
