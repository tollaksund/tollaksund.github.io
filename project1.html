Project Overview

The final physical prototype of the warehouse robot.

Goal and Context:

This project was a hands-on exercise in developing a fully autonomous mobile robot (AMR) for logistics. The primary goal was to design, build, and program a robot capable of navigating a simulated warehouse, detecting specific items, avoiding dynamic obstacles, and delivering the items to a designated packing bay.

The navigation relies on angle-based localization relative to fixed warehouse landmarks, managed by a robust Finite State Machine (FSM). This demonstrated integration of mechanical design, vision systems, and advanced pathfinding algorithms.

Key Technical Systems

Control Logic

Behavior managed by a Finite State Machine (FSM) for reliable task sequencing.

Mechanical Design

Chassis designed in Fusion 360 and fabricated using 3D printing.

Path Planning

Hybrid obstacle avoidance: Potential Fields for smoothing and Bug 2 for close quarters.

Intelligence

Uses a model-based reflex agent for robust real-time decision making.

Navigation and Obstacle Avoidance

The navigation system converts sensor inputs into motor commands, enabling the robot to turn, drive straight, or correct its path. A hybrid method is used: potential fields for smooth long-range movement, and Bug 2 for close-range obstacle avoidance. This allows the robot to maintain a smooth trajectory while dynamically responding to its environment.

  • Bug 2 Algorithm: Follows the edge of an obstacle until a clear path to the goal is found, ensuring reliable obstacle circumvention.
  • Potential Fields: Obstacles act as repelling forces and the goal acts as an attractor, resulting in fluid motion toward the target.

Operational Stages (FSM)

The robot operates using a structured navigation system divided into three main stages, coordinated by the Finite State Machine:

  • Stage 1 – Item Location: The robot scans for the packing bay and row markers. It avoids blind zones and aligns using shelf angles to reach a stable scanning zone.
  • Stage 2 – Item Pickup: It raises the grabber, aligns with the detected item, and drives forward to activate the pick-up mechanism.
  • Stage 3 – Delivery: The robot navigates back toward the packing bay using shelf alignment and vision input. Upon arrival, it drops the item and resets the FSM.

Design, Validation, and Reflection

The electronics were prototyped using an Arduino-based system, and the robot logic was implemented on a Raspberry Pi running a modular navigation system. During real-world tests, the robot successfully retrieved and delivered items with a success rate of approximately 75%. It demonstrated robust integration between navigation, obstacle avoidance, and vision systems.

Skills Developed:

This project helped develop skills in embedded programming, sensor integration, and mechanical design. It also emphasized the importance of structured architecture, real-time decision-making, and robust testing.
